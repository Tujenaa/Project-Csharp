Phân Quyền (RBAC)

🔴
ADMIN
  • Quản lý tất cả POI / Audio / Users
  • Phê duyệt / từ chối POI
  • Xóa hàng loạt POI bị từ chối
  • Xem lịch sử phát toàn hệ thống
  • Xem chi tiết User + POI của Owner
  • Dashboard thống kê tổng quan
  • Badge pending trên sidebar
  • Tạo user bất kỳ role
🟣
OWNER
  • Chỉ thấy POI có OwnerId = mình
  • Tạo POI → PENDING, chờ Admin duyệt
  • Sửa POI APPROVED → về PENDING lại
  • CRUD Audio (chỉ khi POI APPROVED)
  • Upload ảnh POI ≤ 5MB/file
  • Sửa thông tin cá nhân (/Profile)
  • Đăng ký → role=OWNER, vào luôn
🟢
CUSTOMER
  • Chỉ dùng Mobile App Android
  • Bị chặn đăng nhập Web CMS
  • GPS geofencing tự động phát TTS
  • Nghe 4 ngôn ngữ vi/en/ja/zh
  • Offline mode — SQLite + seed JSON
  • Sửa Name + Phone (AccountPage)
  • Lịch sử nghe cá nhân
POI Workflow: Owner tạo → PENDING → Admin duyệt → APPROVED (hiện App). Từ chối → REJECTED + lý do. Owner sửa APPROVED → về PENDING. Header X-Role: ADMIN/OWNER phân biệt tại API.

Chapter 05

Authentication

🌐 Web CMS — Session

HttpContext.Session lưu UserId, Username, Role. CUSTOMER bị chặn. Register → role=OWNER, vào luôn. So sánh password trực tiếp (plaintext).

📱 Mobile — Preferences

AuthService static: lưu token GUID, user_id, username, name, email, phone. Guest mode: UserId=0, không ghi History. SessionService.CurrentUser in-memory.

Bảo mật: Password lưu plaintext trong DB (chưa bcrypt). Chấp nhận được cho đồ án, cần cải thiện cho production.

Chapter 06

GPS Geofencing — Core Feature

Haversine formula tính khoảng cách client-side (R=6,371,000m). Khi distance ≤ Radius → tự động phát TTS → ghi History. Loop 3 giây, không cần server roundtrip.

① Load POI

Online: GET /api/poi → cache SQLite. Offline: GetCachedPOIsAsync(). Lần đầu: seed từ seed_pois.json.

② GPS Tracking

LocationService.GetCurrentLocationAsync(GeolocationAccuracy.Best, 5s)

③ Haversine Check

DistanceHelper.GetDistance(lat1,lon1,lat2,lon2) → mét. Hiển thị "350m" hoặc "1.2km".

④ TTS Playback

GetScriptForLang → TranslateService nếu cần → SpeakAsync + waveform. OnTtsFinished → ghi History.

⑤ Ghi History

Online: POST /api/history — API kiểm tra Role=CUSTOMER. Offline: SQLite PendingHistory → sync khi online.

TrackUserLocation loop 3s → Haversine → TTS → ghi History
▼ 
sequenceDiagram
    participant VM as MapViewModel
    participant LOC as LocationService
    participant DH as DistanceHelper
    participant TTS as TextToSpeechService
    participant TR as TranslateService
    participant HS as HistoryStore
    participant API as TourGuideAPI
    participant LDB as LocalDbService

    Note over VM,API: Khoi dong: InitAsync load POI - online thi GET /api/poi va cache SQLite, offline dung cache hoac seed_pois.json
    VM->>VM: TrackUserLocationAsync loop 3s
    VM->>LOC: GetCurrentLocationAsync(Best, 5s)
    LOC-->>VM: Location (lat, lng)
    VM->>VM: RefreshNearbyOrder - sap xep POI

    loop foreach POI
        VM->>DH: GetDistance(userLat, userLng, poi.Lat, poi.Lng)
        DH-->>VM: distance (Haversine, met)
        alt distance <= poi.Radius AND chua auto-play AND khong dang phat
            VM->>VM: autoPlayedIds.Add(poi.Id)
            alt Online
                VM->>API: GET /api/poi/{id} - script moi nhat
                API-->>VM: POIDto voi scripts
            end
            alt Language = vi
                VM->>TTS: LoadText(ScriptVi)
            else Language khac
                VM->>TR: TranslateAsync(ScriptVi, lang)
                TR-->>VM: translatedText
                VM->>TTS: LoadText(translatedText)
            end
            VM->>TTS: SpeakAsync(token)
            TTS-->>VM: Phat audio native Android
            TTS->>VM: OnFinished callback
            VM->>HS: AddAsync(poi)
            HS->>API: POST /api/history (poiId, userId)
            API-->>HS: OK
        else distance > poi.Radius*2
            VM->>VM: autoPlayedIds.Remove - reset
        end
    end
