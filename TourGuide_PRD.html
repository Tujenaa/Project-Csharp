Chapter 01

Phân Quyền & Vai Trò (RBAC)

3 role với quyền hạn rõ ràng, kiểm soát cả ở tầng Web CMS và API. Role CUSTOMER bị chặn hoàn toàn tại Web CMS — chỉ có thể dùng Mobile App. Mỗi role có giao diện và dữ liệu hiển thị riêng biệt.

🔴
ADMIN
  • Xem & quản lý TẤT CẢ POI toàn hệ thống
  • Xem & quản lý TẤT CẢ Audio scripts
  • Quản lý Users — thêm/sửa/xóa/đổi role
  • Xem toàn bộ lịch sử phát (History)
  • Xem thống kê tổng quan hệ thống
  • Truy cập Web CMS đầy đủ
  • Menu /Users hiển thị và hoạt động
  • /Profile bị ẩn (dùng admin tools)
🟣
OWNER
  • Chỉ thấy POI của mình (filter OwnerId)
  • CRUD đầy đủ POI & Audio của mình
  • Xem lịch sử phát POI mình sở hữu
  • Sửa thông tin cá nhân (/Profile)
  • Preview TTS trực tiếp trong browser
  • Menu /Users bị ẩn hoàn toàn
  • Đăng ký /Register → vào luôn (no approval)
  • Thống kê dashboard riêng của mình
🟢
CUSTOMER
  • Chỉ sử dụng Mobile App (MAUI Android)
  • Bị chặn đăng nhập Web CMS (redirect)
  • GPS geofencing tự động phát audio
  • Nghe TTS đa ngôn ngữ (vi/en/ja/zh)
  • Ghi lịch sử phát — kiểm soát bởi Trigger
  • Xem bản đồ Leaflet & chi tiết POI
  • Cài đặt ngôn ngữ & bán kính geofence
  • Xem lịch sử chuyến đi của mình

DTO Pattern — Bảo vệ thông tin nhạy cảm

POIDto (Mobile App): Không có OwnerId. Chứa 4 cột script trực tiếp (ScriptVi, ScriptEn, ScriptJa, ScriptZh) thay vì nested Audio object. Kèm List<string> Images là danh sách URL ảnh. Mobile App build full URL từ BaseUrl + imageUrl. POI Full (Web CMS): Đầy đủ bao gồm OwnerId, Status, RejectReason. PasswordHash không bao giờ trả qua bất kỳ API response nào.

Chapter 04

Text-to-Speech Pipeline

TTS hoàn toàn miễn phí, không cần API key, không tốn chi phí vận hành. Dùng Web Speech API native của browser/WebView kết hợp Google Translate tự động dịch. Owner chỉ cần viết 1 script tiếng Việt — hệ thống tự dịch và phát 4 ngôn ngữ.

Pipeline Auto-Translate → TTS

📝 Script (tiếng Việt) → 🌐 Google Translate API → 📄 Translated Text → 🎤 pickVoice(lang) → 🔊 SpeechSynthesis.speak()

4 Ngôn Ngữ Được Hỗ Trợ

🇻🇳 vi-VN
Tiếng Việt (Mặc định) Ưu tiên: Microsoft Hoai My Neural → Microsoft Nam Minh Neural → fallback system. Cần cài voice pack trong Windows/Android Settings để có giọng tự nhiên. Đây là ngôn ngữ gốc — không qua dịch.
🇺🇸 en-US
Tiếng Anh Có sẵn trên mọi thiết bị: Microsoft Aria Online (Natural) → Microsoft Guy Online → Google US English. Fallback chain tự động theo thứ tự ưu tiên.
🇯🇵 ja-JP
Tiếng Nhật Microsoft Nanami Online (Natural) → Google 日本語. Chất lượng cao nhất trên Edge/WebView. Script dịch qua Google Translate trước khi phát. Phổ biến với du khách Nhật.
🇨🇳 zh-CN
Tiếng Trung (Phổ thông) Google 普通话 (中国) → Microsoft Xiaoxiao Online Neural. Fallback chain tự động nếu voice không khả dụng trên thiết bị Android cụ thể.

TTS trên Mobile MAUI vs Web Browser

📱 Mobile MAUI TTS

Dùng TextToSpeech.SpeakAsync(text, options) của .NET MAUI. Chọn locale theo ngôn ngữ đã cài trong Settings page. 4 fallback levels nếu locale không khả dụng trên thiết bị. Phát ngầm khi geofence trigger.

🌐 Web Browser TTS

Dùng window.speechSynthesis.speak(utterance) trực tiếp trong browser. Owner preview TTS ngay trong Audio Manager mà không cần cài app. Dùng SpeechSynthesisUtterance với locale phù hợp.

Zero Cost Guarantee: Web Speech API là API native của trình duyệt — không cần server, không cần API key, không tốn tiền. Google Translate dùng free tier không cần key cho translation ngắn. Tổng chi phí vận hành TTS = $0/tháng.

Chapter 06

GPS Geofencing — Core Feature

Tính năng cốt lõi của hệ thống. Khi khách du lịch bước vào bán kính POI, app tự động phát audio thuyết minh đúng ngôn ngữ đã chọn mà không cần bất kỳ tương tác nào. Sử dụng công thức Haversine tính khoảng cách trên thiết bị — không cần server roundtrip.

5 Bước Xử Lý Geofencing

Bước 1 — Khởi động & Tải dữ liệu POI

App login → lưu UserId local → GET /api/poi → tải về danh sách POI APPROVED dạng DTO (tên, tọa độ, Radius, scripts 4 ngôn ngữ, images). Lưu vào bộ nhớ app để tính offline.

Bước 2 — GPS Tracking liên tục

Bắt đầu Geolocation.GetLocationAsync() với DesiredAccuracy = High. Lọc nhiễu bằng accuracy threshold. Cập nhật vị trí liên tục và hiển thị trên bản đồ Leaflet.

Bước 3 — Haversine Distance Check

Với mỗi POI trong danh sách: tính khoảng cách từ vị trí hiện tại đến tọa độ POI bằng công thức Haversine (tính trên mặt cầu, đơn vị mét). So sánh với Radius của POI.

Bước 4 — Trigger Audio Playback

Distance ≤ Radius → chọn Audio script theo ngôn ngữ đã cài trong Settings → Google Translate dịch (nếu cần) → TTS phát via TextToSpeech.SpeakAsync(). Cooldown timer để không phát lại ngay lập tức.

Bước 5 — Ghi History & UI Update

Sau khi phát xong → POST /api/history với {UserId, PoiId} → HistoryController kiểm tra User tồn tại và Role = CUSTOMER → INSERT thành công. Guest (UserId=0) bị từ chối, không ghi history. UI hiển thị popup tên POI, tap để xem chi tiết.

⚡ Haversine Formula

Tính khoảng cách trên mặt cầu trái đất — chính xác hơn Euclidean distance. Được implement client-side trong C# MAUI. Không cần server roundtrip mỗi lần GPS update → tiết kiệm pin và data.

⏱️ Cooldown Timer

Sau khi phát audio cho một POI, hệ thống set cooldown (mặc định có thể configure). Tránh phát lại khi user đứng tại chỗ hoặc di chuyển ra/vào bán kính liên tục. Reset khi user rời xa hơn 2× radius.

🎯 POI Marker Priority & Background

Phát hiện khi nhiều vùng POI overlap → Hệ thống lọc và ưu tiên tự động phát POI gần nhất với User. Đã có support nhận biết Geofence kể cả trong Background Mode liên tục.

🔋 Guest Offline & History

Khách vãng lai (Guest) không cần mạng vẫn sử dụng bình thường. History không lưu cho Guest để bảo mật. Tích hợp thông báo UI khi call API bị lỗi do mất mạng.

📡 Sequence — Geofence Trigger
sequenceDiagram
    participant C as Customer (Phone)
    participant APP as TourGuideApp (MAUI)
    participant GEO as GeoService
    participant TTS as TextToSpeechService
    participant API as TourGuideAPI
    participant DB as SQL Server

    C->>APP: Mở App → Login thành công
    APP->>API: GET /api/poi
    API->>DB: SELECT POI (APPROVED) JOIN Audio ON PoiId
    DB-->>API: POI[] with Audio scripts (vi/en/ja/zh columns)
    API-->>APP: List~POIDto~ (POIs loaded)
    APP->>GEO: StartTracking()
    GEO->>GEO: Geolocation.GetLocationAsync() loop

    loop GPS Update (every ~3s)
        GEO->>GEO: getCurrentPosition() → {lat, lng, accuracy}
        alt accuracy > 50m → skip (GPS noise)
            GEO-->>APP: Skip update
        else accuracy ≤ 50m → process
            GEO->>GEO: foreach POI: HaversineDistance(userPos, poi)
            alt Distance ≤ POI.Radius (e.g. 30m)
                GEO-->>APP: OnGeofenceEnter(poi)
                APP->>APP: Check cooldown timer — phát lại chưa?
                alt Trong cooldown (5 phút)
                    APP-->>C: Không phát lại
                else Hết cooldown / lần đầu
                    APP->>APP: FindAudio(poi, userLanguage) — đọc cột vi/en/ja/zh
                    alt Audio script tồn tại cho ngôn ngữ
                        APP->>TTS: SpeakWithTranslation(script, lang)
                        TTS->>TTS: Google Translate (nếu lang ≠ vi)
                        TTS->>TTS: TextToSpeech.SpeakAsync(translated, locale)
                        TTS-->>C: 🔊 Audio thuyết minh phát
                        APP-->>C: Popup "Bạn đang tại: [POI Name]"
                    else Không có script ngôn ngữ này
                        APP->>TTS: SpeakAsync(poi.Description, lang)
                        TTS-->>C: 🔊 Phát mô tả POI
                    end
                    APP->>APP: SetCooldown(poi.Id, 5min)
                    APP->>API: POST /api/history {UserId, PoiId}
                    API->>API: Kiểm tra User.Role = CUSTOMER
                    API->>DB: INSERT INTO History (PlayTime = now)
                    DB-->>API: 200 OK (inserted)
                    API-->>APP: History saved ✅
                end
            end
        end
    end

🎙️ Sequence — TTS Pipeline
sequenceDiagram
    participant APP as TourGuideApp
    participant TTS as TextToSpeechService
    participant GT as Google Translate API
    participant MAUI as MAUI TextToSpeech
    participant C as Customer

    APP->>TTS: TranslateAndSpeak(script="Đây là...", targetLang="en")

    TTS->>TTS: sourceLanguage = "vi" (script gốc tiếng Việt)
    alt targetLang == sourceLanguage (vi → vi)
        TTS->>TTS: Skip translation
        TTS->>TTS: translatedText = script
    else targetLang ≠ sourceLanguage (vi → en/ja/zh)
        TTS->>GT: POST translate {text=script, from=vi, to=en}
        GT-->>TTS: translatedText = "This is..."
    end

    TTS->>TTS: GetLocale(targetLang)
    Note over TTS: en → "en-US", ja → "ja-JP", zh → "zh-CN"

    TTS->>MAUI: GetAvailableLocalesAsync()
    MAUI-->>TTS: availableLocales[]

    TTS->>TTS: PickBestVoice(locales, targetLocale)
    alt Preferred voice available (e.g. en-US)
        TTS->>TTS: options.Locale = en-US
    else Partial match (e.g. en-GB)
        TTS->>TTS: options.Locale = en-GB (fallback)
    else No match
        TTS->>TTS: options.Locale = default system locale
    end

    TTS->>MAUI: SpeakAsync(translatedText, options)
    MAUI-->>C: 🔊 Audio plays on phone speaker

    alt TTS fails (locale error)
        TTS->>TTS: Retry với default locale
        TTS->>MAUI: SpeakAsync(translatedText, defaultOptions)
        MAUI-->>C: 🔊 Fallback audio plays
    end

    MAUI-->>TTS: SpeakAsync completed
    TTS-->>APP: Speaking finished
    APP->>APP: POST /api/history {UserId, PoiId} (record playback)

End-to-End Flow — Luồng Hoàn Chỉnh

Từ khi Owner tạo POI trên Web CMS, đến khi Customer nghe thuyết minh trên Mobile App — toàn bộ chu trình của một điểm tham quan.

sequenceDiagram
    participant O as Owner (Web CMS)
    participant WEB as GPSGuide.Web
    participant API as TourGuideAPI
    participant DB as SQL Server
    participant APP as TourGuideApp (MAUI)
    participant C as Customer

    Note over O,DB: Giai đoạn 1 — Owner tạo POI & Audio
    O->>WEB: POST /Pois (Name, Lat, Lng, Radius, Address)
    WEB->>API: POST /api/poi {Status=PENDING, OwnerId=...}
    API->>DB: INSERT INTO POI
    DB-->>API: POI Id = 42
    API-->>WEB: 200 OK
    WEB-->>O: ✅ POI tạo thành công — chờ Admin duyệt

    O->>WEB: POST /AudioManager (PoiId=42, vi=..., en=..., ja=..., zh=...)
    WEB->>API: POST /api/audio {PoiId=42, vi=..., en=..., ja=..., zh=...}
    API->>DB: INSERT INTO Audio (1 row, 4 columns)
    DB-->>API: Audio record
    API-->>WEB: 200 OK
    WEB-->>O: ✅ Script đã lưu

    Note over O,WEB: Owner preview TTS
    O->>WEB: Click "Preview TTS"
    WEB->>WEB: Web Speech API.speak(Script.vi, vi-VN)
    WEB-->>O: 🔊 Nghe thử tiếng Việt

    Note over O,DB: Giai đoạn 1b — Admin duyệt POI
    O->>WEB: Admin vào /Approval — duyệt POI 42
    WEB->>API: PUT /api/poi/42/approve
    API->>DB: UPDATE POI SET Status='APPROVED'
    DB-->>API: 200 OK
    API-->>WEB: POI approved

    Note over APP,DB: Giai đoạn 2 — Customer khởi động App
    C->>APP: Mở TourGuideApp
    APP->>API: POST /api/users/login {username, password}
    API->>DB: SELECT * FROM Users WHERE Username=?
    DB-->>API: User (Role=CUSTOMER)
    API-->>APP: {Id, Username, Name, Email, Phone, Role}
    APP->>API: GET /api/poi
    API->>DB: SELECT POI (APPROVED) JOIN Audio → flatten vi/en/ja/zh
    DB-->>API: POI[] dạng DTO (ScriptVi, ScriptEn, ScriptJa, ScriptZh, Images[])
    API-->>APP: POI list đầy đủ
    APP->>APP: Khởi động GPS tracking (Geolocation API)

    Note over APP,DB: Giai đoạn 3 — Geofence Trigger
    loop Mỗi GPS update
        APP->>APP: Haversine(userPos, POI[42]) → dist
        alt dist ≤ Radius (e.g. 30m)
            APP->>APP: pickScript(poi=42, lang=vi) → ScriptVi
            APP->>APP: Google Translate (nếu lang ≠ vi)
            APP->>APP: TextToSpeech.SpeakAsync(script)
            APP-->>C: 🔊 Thuyết minh tự động phát
            APP->>API: POST /api/history {UserId, PoiId=42}
            API->>API: Kiểm tra User.Role = CUSTOMER
            API->>DB: INSERT INTO History (PlayTime=now)
            DB-->>API: 200 OK
            API-->>APP: History saved
            APP-->>C: Popup "Bạn đang tại: [POI Name]"
        end
    end
