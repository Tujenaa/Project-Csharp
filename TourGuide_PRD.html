Chapter 03

Phân Quyền & Vai Trò (RBAC)

Hệ thống có 3 role cố định, được lưu trong cột Role bảng Users với CHECK constraint ('ADMIN','OWNER','CUSTOMER'). Role được kiểm tra tại cả tầng Web CMS (session) và API (header X-Role).

🔴
ADMIN
  • Đăng nhập Web CMS (session-based)
  • Xem tất cả POI (/api/poi/all)
  • Xem POI pending (/api/poi/pending)
  • Approve/Reject POI (PUT .../approve, .../reject)
  • Sửa POI — status giữ nguyên (không về PENDING)
  • Xóa POI bị reject hàng loạt (DELETE /rejected)
  • Xem tất cả Audio, History, Users
  • CRUD Users — thêm/sửa/xóa/đổi role
  • Badge đếm POI pending trên navbar
  • Dashboard thống kê toàn hệ thống
  • Menu /Users hiển thị và hoạt động
  • Menu /Profile ẩn
🟣
OWNER
  • Đăng nhập Web CMS
  • Chỉ thấy POI của mình (OwnerId = session UserId)
  • Tạo POI → status PENDING (chờ Admin duyệt)
  • Sửa POI của mình → về PENDING lại
  • Xóa POI của mình (kiểm tra OwnerId)
  • CRUD Audio cho POI của mình
  • Preview TTS trực tiếp trong browser
  • Upload ảnh POI (tối đa 5 ảnh/lần, 5MB/ảnh)
  • Đặt ảnh thumbnail
  • Đăng ký /Register → auto login OWNER
  • Sửa thông tin cá nhân /Profile
  • Dashboard thống kê chỉ POI mình
  • Menu /Users ẩn hoàn toàn
🟢
CUSTOMER
  • Đăng nhập Mobile App (POST /api/users/login)
  • Bị chặn đăng nhập Web CMS (thông báo lỗi rõ ràng)
  • Tự đăng ký qua App (POST /api/users/register)
  • Đăng nhập offline dạng guest
  • Xem bản đồ Leaflet + markers POI
  • GPS geofencing tự động phát TTS
  • Nghe TTS 4 ngôn ngữ vi/en/ja/zh
  • Tìm kiếm POI trên bản đồ
  • Vẽ lộ trình đến POI
  • Xem chi tiết POI (PlaceDetailPage)
  • Xem lịch sử chuyến đi (/api/history/user/{id})
  • Sửa tên và SĐT cá nhân
  • Cài đặt ngôn ngữ TTS, auto-play, GPS

Web Auth Middleware (Program.cs): Tất cả request không phải /Auth, /Register, /img-proxy đều kiểm tra session["UserId"]. Nếu null → redirect /Auth. CUSTOMER bị chặn tại bước đăng nhập (OnPostAsync) trước khi session được set.
API — không có auth middleware: TourGuideAPI không dùng JWT hay middleware auth. Phân quyền business-logic được xử lý thủ công qua header X-Role (gửi từ Web CMS khi cần) và kiểm tra user.Role trong code controller. Mobile App gửi userId trong body; API kiểm tra user.Role == "CUSTOMER".

DB Trigger: trg_OnlyCustomerHistory

INSTEAD OF INSERT trên bảng History: Trigger kiểm tra role của UserId. Nếu u.Role <> 'CUSTOMER' → RAISERROR, không insert. Đây là lớp bảo vệ cuối ở tầng DB, ngay cả khi bỏ qua tầng API.

Lưu ý trong SQL file: Có lệnh DROP TRIGGER trg_OnlyCustomerHistory ngay sau CREATE — cần bỏ dòng này khi deploy thật.

EF Core Model Notes

Các trường [NotMapped]: POI.OwnerName và POI.ImageUrl được đánh dấu [NotMapped] — chỉ dùng trong DTO logic, không có trong DB. ImageUrl được set từ bảng POIImages (ảnh thumbnail đầu tiên) trong API response hoặc Web model.
Audio.POI và History.POI có [JsonIgnore] để tránh vòng lặp khi serialize JSON.

PendingCountFilter (IAsyncPageFilter)

Đăng ký global qua AddMvcOptions. Trước mỗi page handler execution: nếu role=ADMIN → gọi GET /api/poi/pending → ghi ViewData["PendingCount"] → hiển thị badge trên navbar.

Route đặc biệt không qua Razor Pages. Web CMS dùng để relay ảnh từ API server (vì ảnh lưu trên wwwroot của API, không phải Web). Nhận param url, prepend ApiBaseUrl nếu là relative path, trả về bytes với đúng MIME type.

06.3

Quản lý Audio (AudioManager.cshtml)

Mỗi POI tối đa 1 bản ghi Audio với 4 ngôn ngữ (vi/en/ja/zh). Owner chỉ thấy Audio của POI mình. Admin thấy tất cả.

Ràng buộc tạo Audio

API kiểm tra 3 điều kiện trước khi tạo Audio:
  1. POI tồn tại
  2. POI status phải là APPROVED, PENDING_EDIT, hoặc PENDING_DELETE (không phải PENDING thuần)
  3. POI chưa có Audio (unique 1:1)

Tính năng Preview TTS

Web CMS hỗ trợ preview TTS trực tiếp trong browser thông qua Web Speech API (JavaScript). Không cần gọi server — browser native speech synthesis phát trực tiếp text trong form. Owner có thể nghe thử trước khi lưu.

Google Translate tích hợp trên Web

AudioManager.cshtml có nút "Dịch tự động" — gọi translate.googleapis.com/translate_a/single từ phía JavaScript (frontend) để điền nhanh các ô en/ja/zh từ nội dung tiếng Việt. Owner chỉ cần viết vi, hệ thống tự dịch sang 3 ngôn ngữ còn lại.

Luồng Duyệt POI (Approval)

Workflow 3 trạng thái: PENDING → APPROVED hoặc REJECTED. Owner tạo/sửa POI → PENDING. Admin approve/reject.

APPROVEDAdmin đã duyệt, đang hoạt độngTất cả✅ Có
REJECTEDAdmin từ chối, có RejectReasonOwner (POI của mình) + Admin❌ Không
Owner tạo POI → PENDING
→

→

Ngoài ra có LoginPage (trước Shell) và PlaceDetailPage (navigation từ MapPage qua Shell.GoToAsync("placeDetail") với QueryProperty poi).

Services Architecture

ApiService

Singleton HTTP client (timeout 10s). Tự động chọn BaseUrl theo DeviceType. Online-first → cache SQLite → offline fallback. Sync pending history khi online.

LocalDbService (Singleton)

SQLite database tại AppDataDirectory. 2 bảng: POICache (offline POI) và PendingHistory (queue chờ sync). Tự seed từ seed_pois.json nếu DB trống.

AuthService (Static)

Lưu session qua MAUI Preferences. Keys: user_token, user_id, user_username, user_name, user_email, user_phone. Hỗ trợ offline guest mode.

SettingService (Singleton)

Lưu Preferences: app_language (vi/en/ja/zh), auto_play (bool), gps_enabled (bool). INotifyPropertyChanged để binding UI.

LocationService

MAUI Geolocation.GetLastKnownLocationAsync() → GetLocationAsync(). Trả về Location (Latitude, Longitude).

TextToSpeechService (Partial)

Platform-specific: Android native TTS (TextToSpeechService.Android.cs) với Progress/Finished events. MAUI fallback cho iOS/Windows.

MapService

BuildLeafletHtml() — tạo HTML string nhúng Leaflet.js + OpenStreetMap. Giao tiếp 2 chiều qua EvaluateJavaScriptAsync() và URL scheme tourguide://poi/{id}.

TranslateService

Gọi translate.googleapis.com free endpoint. User-Agent bắt buộc để không bị 403. Retry wrapper tối đa 2 lần. Fallback text gốc nếu lỗi.

ConnectivityService

MAUI Connectivity.Current.NetworkAccess wrapper. Kiểm tra trước mọi API call trong ApiService.

GPS Geofencing & Auto-play

MapViewModel chạy vòng lặp TrackUserLocationAsync() trong background — poll GPS mỗi 3 giây và kiểm tra khoảng cách đến từng POI.

Logic Geofencing

// Mỗi 3 giây:
loc = await locationService.GetCurrentLocationAsync();
→ setUserLocation(lat, lon) qua JS bridge
→ RefreshNearbyOrder() — sắp xếp POI theo khoảng cách gần nhất
→ CheckNearbyPOI(lat, lon)

// CheckNearbyPOI:
foreach poi in NearbyPOI:
    dist = DistanceHelper.GetDistance(userLat, userLon, poi.Lat, poi.Lon)
    
    // Reset: nếu đã ra xa hơn 2x radius → xóa khỏi autoPlayedIds
    if dist > poi.Radius * 2 && autoPlayedIds.Contains(poi.Id):
        autoPlayedIds.Remove(poi.Id)
    
    // Trigger: chưa auto-play AND trong radius AND không đang phát
    if dist < poi.Radius && !autoPlayedIds.Contains(poi.Id) && !isPlaying:
        chọn POI gần nhất trong số các POI hợp lệ

→ autoPlayedIds.Add(closestPoi.Id)
→ HandlePlay(closestPoi) // chỉ phát nếu SettingService.AutoPlay = true
Điều kiện auto-play: (1) SettingService.Instance.AutoPlay == true, (2) SettingService.Instance.GpsEnabled == true, (3) khoảng cách < poi.Radius (mét), (4) POI chưa trong autoPlayedIds, (5) không đang phát TTS khác.

Lịch Sử Phát (History)

Khi nào ghi History?

History được ghi khi TTS phát xong — sự kiện OnTtsFinished. Không ghi khi bắt đầu phát hoặc khi pause. Đây là thiết kế có chủ đích: chỉ tính lần nghe hoàn chỉnh.

HistoryStore (static helper)

HistoryStore.AddAsync(poi) lấy userId từ Preferences.Get("user_id", 0) rồi gọi ApiService.SaveHistory(poi.Id, userId). Nếu userId=0 (guest) → API sẽ từ chối với lỗi "UserId bị null".

Luồng bảo vệ 2 tầng

App: POST /api/history {poiId, userId}
→
API: kiểm tra userId != 0, user tồn tại, role=CUSTOMER
→
DB Trigger: INSTEAD OF INSERT — kiểm tra lại role trong Users table
→
INSERT vào History

History trên App (HistoryPage)

Gọi GET /api/history/user/{userId} — trả về list sắp xếp PlayTime DESC, kèm PoiName và PoiImage. Hiển thị timeline chuyến đi của user. Chỉ hoạt động khi online.

History trên Web CMS

Admin thấy tất cả history toàn hệ thống. Owner chỉ thấy history của POI mình. Dashboard thống kê: top 5 POI được nghe nhiều nhất (group by PoiName, count).

Dịch Tự Động — Google Translate Free

Hệ thống dùng Google Translate free endpoint (không cần API key) ở cả 2 phía: Web CMS (JavaScript frontend) và Mobile App (C# TranslateService).

Endpoint sử dụng

GET https://translate.googleapis.com/translate_a/single
  ?client=gtx
  &sl=vi        (source: luôn là tiếng Việt)
  &tl={toLang}  (en / ja / zh)
  &dt=t         (chỉ lấy translation)
  &q={encoded_text}
User-Agent bắt buộc: TranslateService.cs phải set header User-Agent: Mozilla/5.0... — nếu không Google trả 403 Forbidden.

Response parsing

// Response format: [[ ["dịch","gốc",...], ... ], ...]
// Nối tất cả segment [i][0] thành chuỗi hoàn chỉnh
var segments = doc.RootElement[0];
foreach seg in segments: sb.Append(seg[0].GetString());

Fallback strategy

  • Nếu toLang = "vi" → trả ngay text gốc, không gọi mạng
  • Nếu exception → trả text gốc tiếng Việt (fallback)
  • RetryAsync: tối đa 2 lần với delay 1s giữa các lần
  • App: nếu ScriptEn/Ja/Zh null → fallback về ScriptVi → fallback về Description

Chapter 11

Giới Hạn & Ràng Buộc Kỹ Thuật

🔒 Bảo mật chưa hoàn chỉnh

  • Password lưu plain text (không bcrypt)
  • Web Auth gọi GET /api/users (expose toàn bộ users list)
  • API không có JWT / bearer token
  • CORS AllowAll (AllowAnyOrigin)
  • X-Role header có thể giả mạo từ bên ngoài

📦 Audio giới hạn

  • Mỗi POI chỉ có 1 bản ghi Audio duy nhất
  • Chỉ tạo Audio khi POI status ≠ PENDING thuần
  • Google Translate có thể bị rate-limit nếu text quá dài
  • TTS Android: cần voice pack cài sẵn trên thiết bị

🌐 Network

  • IP API hard-code 192.168.1.83 (physical device)
  • App timeout 10s — có thể thất bại trên mạng chậm
  • Image phải proxy qua /img-proxy (cross-origin)
  • Leaflet cần internet để load OpenStreetMap tiles

🖼️ Upload ảnh

  • Max 5MB per file
  • Chỉ jpg/jpeg/png/webp
  • Ảnh lưu trên wwwroot của API server (không phải CDN)
  • Mỗi lần upload: max 5 file (lấy 5 đầu tiên)

📱 Platform

  • Target chính: Android (emulator + physical)
  • iOS/Windows/Tizen scaffold có nhưng chưa test
  • TTS native: chỉ có partial implementation Android
  • SQLite path: FileSystem.AppDataDirectory

🗄️ Database

  • SQL file có lệnh DROP TRIGGER (cần xóa khi deploy)
  • Status DEFAULT = 'APPROVED' (ALTER TABLE) — POI cũ không có cột Status
  • Windows Auth only (Trusted_Connection) — không dùng được SQL Auth
  • EF Core code-first nhưng migration qua SQL script thủ công
